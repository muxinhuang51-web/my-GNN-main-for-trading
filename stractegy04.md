# 从模型到选股回测的打通计划

## 1. 背景与目标
- **当前状态**：已有一个训练好的 RGCN 回归模型，可基于当日异构图输出所有股票的下一交易日收益率预测值。
- **目标**：将模型的预测值转化为可执行的股票组合，并通过历史回测验证其选股能力，最终形成端到端的量化策略原型。
- **总耗时预估**：约 6 小时（可分两天完成）。

---

## 2. 核心链路
```
每个交易日 T：
  1. 构建 T 日的异构图（仅使用 T-1 及之前的信息）
  2. 用训练好的模型预测所有股票的 T+1 收益率
  3. 根据预测值排序，选择 Top-K 股票构成多头组合
  4. 计算 T+1 日该组合的真实收益率（等权）
  5. 逐日滚动，形成策略收益序列
  6. 评估：年化收益、夏普、最大回撤、IC、分组收益等
```

---

## 3. 任务分解

### 3.1 预测生成模块
**目标**：编写函数，输入一个日期，输出该日期所有股票的预测收益率。

**函数要求**：
```python
def predict_on_date(date, model, graph_builder, feature_config):
    """
    参数:
        date: str or datetime, 当前决策日期
        model: 已加载的训练好的 StockRGCN_Regression
        graph_builder: 可构建 HeteroData 的对象或函数
        feature_config: 特征处理所需参数（如标准化用的均值/标准差）
    返回:
        pred_df: pd.DataFrame, columns=['stock_code','pred_return']
    """
```

**实现要点**：
- 使用 `date` 当天能获取的行情、行业、概念等数据构建图（所有信息截止到 `date - 1` 交易日）。
- 特征的 z-score 标准化必须使用训练集统计量（均值 μ、标准差 σ）。
- 模型设为 `eval` 模式，用 `torch.no_grad()` 进行推理。
- 返回的 DataFrame 需与原始股票代码对应，方便后续选股。

**产出**：`prediction.py` 模块，包含 `predict_on_date()` 函数。

---

### 3.2 选股与组合构建模块
**目标**：根据预测值生成实际持仓列表。

**选股规则（第一版建议）**：
- **Top-50 等权**：取预测收益率最高的 50 只股票，等权重配置。
- 剔除当日停牌或 ST 股票（需接入股票状态表）。
- 若不足 50 只（如大量停牌），则取实际数量。

**函数示例**：
```python
def select_stocks(pred_df, top_n=50, exclude_status=True):
    """
    pred_df: pd.DataFrame, columns=['stock_code','pred_return']
    top_n: int, 选择数量
    exclude_status: bool, 是否剔除不可交易股票
    返回: list of stock_code
    """
```

**后续可扩展**：
- 行业中性：在每个行业内选 Top-K，保持行业分布与基准一致。
- 市值加权或预测值加权。
- 加入风险约束（如最大个股偏离、最大行业偏离）。

**产出**：`selection.py` 模块，包含选股逻辑。

---

### 3.3 回测引擎
**目标**：逐日滚动，模拟交易，生成策略每日收益序列。

**主循环框架**：
```python
dates = sorted(list_of_trading_dates)  # 回测区间内的所有交易日
daily_portfolio_ret = []

for i, current_date in enumerate(dates):
    if i == len(dates)-1:  # 最后一天无法计算下日收益
        break
    next_date = dates[i+1]
    
    # 1. 预测
    pred_df = predict_on_date(current_date, model, graph_builder, config)
    # 2. 选股
    selected = select_stocks(pred_df, top_n=50)
    # 3. 计算下个交易日等权收益
    next_ret = get_portfolio_return(selected, next_date)
    daily_portfolio_ret.append(next_ret)
```

**`get_portfolio_return` 细节**：
- 取 `next_date` 这些股票的实际日收益率（百分比收益率或对数收益率）。
- 等权平均（或按预设权重）。
- 处理停牌：无法买入的股票收益记为 0，或从组合中剔除（权重重新归一）。
- 处理涨跌停：如果 `next_date` 开盘一字涨停，则无法买入，收益记为 0；若一字跌停则无法卖出，但这里是做多，可在买入时避开跌停股。

**需要的数据**：
- 全量股票日收益率矩阵，日期对齐。
- 股票每日状态（停牌、ST、涨跌停等）——第一版可先忽略涨跌停，只剔除停牌。

**产出**：`backtest.py`，生成 `portfolio_returns` 序列和基准收益序列（如中证全指等权）。

---

### 3.4 绩效评估模块
**目标**：计算策略的核心评价指标。

**必须指标**：
- **累计收益曲线**：`cum_ret = (1 + daily_ret).cumprod()`
- **年化收益率**：`(cum_ret[-1] ^ (252/len(daily_ret))) - 1`
- **年化波动率**：`daily_ret.std() * sqrt(252)`
- **夏普比率**：年化收益率 / 年化波动率（假设无风险利率为 0）
- **最大回撤**：净值曲线的最大阶段跌幅
- **日度 IC**：每日 `corr(pred_return, actual_return)` 的序列
  - **IC Mean**：IC 序列均值
  - **ICIR**：IC Mean / IC Std（信息系数信息比）
- **分组收益**：每日将预测值分为 5 组（如 Q1 最高组，Q5 最低组），计算各组等权收益，看 Q1 是否显著优于 Q5。

**可视化**：
- 策略 vs 基准净值曲线
- IC 序列图
- 分组收益柱状图

**工具**：可手写计算，也可用 `empyrical` 或 `quantstats` 辅助。

**产出**：`evaluation.py`，以及图表和指标汇总报告。

## 5. 关键注意事项
- **未来信息泄漏**：预测时所有数据（特征、边）仅使用 `date` 之前的信息。标签（下日真实收益）绝对不能参与预测，只在回测评估时计算绩效。
- **停牌处理**：选股时剔除当前处于停牌的股票；若持仓期间发生停牌，收益按 0 或实际复牌后收益处理。
- **涨跌停限制**：若模型选出的股票在下一交易日一字涨停（无法买入），将其收益记为 0；跌停虽不影响多头买入，但最好避免选择可能连续跌停的股票（可通过状态过滤）。
- **幸存者偏差**：股票池必须包含历史各时期实际可交易的全部股票（含已退市），否则会高估收益。
- **成本考虑**：第一版可忽略交易成本（冲击成本、手续费），后续迭代时加入。
